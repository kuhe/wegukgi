June 10th, 2015
Baseball is Hard

I'm willing to admit that I write JavaScript professionally ... You know who created my job career? C++. And before that C and various assembly and electronics and the rise of civilization and the invention of agriculture. A storied lineage. I took a look at the source code for how to plant corn-- I mean, the PHP interpreter and the V8 compilers, and well, what was it all for?

It was probably very fun/interesting in its own right, plus now we can write some really terrible (powerful (terrible (powerful))) code such as eval, and switch types on the fly. What I code is pretty much plain English in comparison to what came before.

Baseball remains the same. I recently added batted ball trajectory animation to my baseball distraction project. I used arctangent in real life, and felt a solemn ghost reach out from the before time and hand me my 10th grade trigonometry book. SOHCAHTOA, it chanted repeatedly. As for the differential equations describing air resistance on a batted ball, well... I just estimated some scalar approximations, womp womp. Take a a few swings sometime.

July 15th, 2014
As I once was

I studied mathematics in college at an age when I had an urge to be book-smart. The last few years has seen that urge all but die out, as I joined the workforce as a code monkey.

I feel it returning. In practice the code only needs to work correctly, but I have a desire to write it in a way that makes it agile to change, decoupled, elastic, readable, and maintainable. And instructive. And I want to be able to do this quickly and efficiently.

Thus I have somehow gotten it in my head that I must review some core CS, namely the topics of data structures and algorithms.
